Site Development

1. Features of a good website:

  • Performance
    • Fast-loading pages: Optimizing image sizes, combing code into a central CSS or JavaScript file as it reduces HTTP requests
    • Easy navigation: Creating a logical page hierarchy using bread scrums, and designing clickable buttons
    • UX: Allowing responsive and adaptive design for all screen sizes
  • Visual Design
    • Consistency in website design in terms of fonts, sizes, headings, sub-headings, and button styles
    • Good typography for clean and easy-to-read design, making the website appealing and easy to comprehend
    • Use of complementary color palette to enhance users’ experience

2. Accessibility considerations:

  • Increased perceivability
    • Text alternatives provided for any non-text content so that it can be changed into other forms people need, such as large print, braille, speech, symbols or simpler language
    • Separated background and foreground for easy reading or hearing
  • Improved operability
    • All functionality available from keyboard
    • No flashing or quick-moving text that could induce epilepsy

3. General Data Protection Regulation:

  • Description

    • General Data Protection Regulation (GDPR). It is a EU data privacy and protection regulation officially enforced on May 25, 2018. It is designed to provide better protection of personal data or personally identifiable information by eliminating personal data breaches and establishing a need to receive consent from a user before collecting and using their data. 7 main principles of GDPR are listed below:

    1. Lawfulness, fairness and transparency
    2. Purpose limitation
    3. Data minimisation
    4. Accuracy
    5. Storage limitaion
    6. Integrity and confidentiality
    7. Accountability
  • Implementation suggestions
    • Website design:

      • Data protection involves website developing with minimization in the amount of personal data required both on the front end and back end. Data should be deleted, either automatically or through user actions, when it is no longer needed. Third parties whom data is passed on such as cloud services would receive requests for data removal, and verifications would be made.

    • Consent:

      • Users would be informed about the data flows containing their information body by providing better consent mechanisms and user controls. The page would provide control over consent settings through user dashboards and account settings.

    • Testing:

      • Testing procedures would be made to predict the ways unauthorized users would access data on this system. External alerts from public to data breach would be implemented.

    4. Website marketing:

    1. Conduct Search Engine Optimisation
    2. Promote webpage using social media
    3. Option for subscription, which sends updates and news of site to user email regularly

    5. Steps for e-commerce:

    1. Purchase one or more domain names for multiple country operation from hosting companies
    2. Obtain secure SSL certificate to secure communications between a visitor’s web browser and the web server hosting your website
    3. Select payment service provider such as Paypal. Linking the checkout page to their payment link, security risks could be minimised
    4. Select shipping courier such as Fedex. Set shipping price standards from package size and weight for smooth checkout
